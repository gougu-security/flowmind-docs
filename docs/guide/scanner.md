# 扫描器

扫描器提供自动化安全检测能力，通过规则引擎和插件系统识别潜在的安全问题。

## 功能概述

- **被动扫描**：自动对捕获的流量进行安全检查
- **内置规则**：9 条预置安全规则
- **插件扩展**：支持 WASM 和声明式插件
- **项目配置**：按项目启用/禁用规则

## 扫描类型

### 被动扫描

被动扫描在流量捕获时自动触发：

```mermaid
flowchart LR
    Traffic[流量捕获] --> Engine[扫描引擎]
    Engine --> Rules[规则匹配]
    Rules --> Findings[安全发现]
```

触发时机：
- HTTP/HTTPS 请求完成后
- WebSocket 会话结束后

### 主动扫描

主动扫描由用户手动触发：

- 对选中的 Flow 进行深度扫描
- 使用 AI 进行安全分析

## 内置规则

| 规则 ID | 名称 | 检测内容 | 严重程度 |
|---------|------|----------|----------|
| `sensitive.info` | 敏感信息泄露 | API Key、Token、密码等 | High |
| `cookie.security` | Cookie 安全 | Secure、HttpOnly、SameSite 属性 | Medium |
| `transport.security` | 传输安全 | HTTP 明文、弱 TLS 配置 | Medium |
| `info.disclosure` | 信息泄露 | 服务器版本、内部路径 | Low |
| `cors.misconfig` | CORS 配置 | 过于宽松的 CORS 策略 | Medium |
| `content.type` | Content-Type | 缺失或错误的 Content-Type | Low |
| `csp.missing` | CSP 缺失 | 缺少 Content-Security-Policy | Low |
| `cert.issue` | 证书问题 | 证书过期、自签名 | Medium |
| `websocket.security` | WebSocket 安全 | 缺少认证、明文传输 | Medium |

## 插件系统

### WASM 插件

使用 WebAssembly 编写的扫描插件：

- **优势**：高性能、跨平台、沙箱隔离
- **语言**：Rust、C/C++、AssemblyScript 等
- **限制**：无文件系统、无网络访问

详见 [WASM 插件开发](../dev/plugins/wasm.md)

### 声明式插件

使用 YAML/JSON/TOML 声明的扫描规则：

- **优势**：易于编写、无需编译
- **适用**：简单的模式匹配规则
- **格式**：支持多种配置格式

详见 [声明式插件开发](../dev/plugins/declarative.md)

## 扫描配置

### 规则管理

进入 **扫描器** → **配置**：

1. 查看所有可用规则
2. 启用/禁用特定规则
3. 保存配置到当前项目

### 插件管理

进入 **扫描器** → **插件**：

1. 查看已安装的插件
2. 导入新插件（ZIP 或目录）
3. 重载插件（修改后刷新）

## 查看结果

### Finding 列表

进入 **扫描器** → **发现**：

| 字段 | 说明 |
|------|------|
| 严重程度 | High / Medium / Low / Info |
| 规则名称 | 触发的规则或插件名称 |
| 描述 | 发现的安全问题描述 |
| 关联 Flow | 触发发现的请求 |
| 时间 | 发现时间 |

### Finding 详情

点击 Finding 查看详情：

- **问题描述**：详细的安全问题说明
- **证据**：触发规则的具体内容
- **修复建议**：如何修复该问题
- **关联数据**：相关的请求/响应数据

## AI 增强扫描

### AI 安全扫描

使用 AI 对 Flow 进行深度分析：

1. 选择要扫描的 Flow
2. 点击 **AI 扫描**
3. AI 分析请求/响应内容
4. 生成安全发现

### 工作流扫描

AI 驱动的工作流级别扫描：

1. 分析 API 调用序列
2. 识别业务逻辑漏洞
3. 生成攻击路径建议

## 使用建议

### 被动扫描最佳实践

1. **保持规则启用**：默认规则覆盖常见问题
2. **定期检查 Findings**：及时处理安全发现
3. **项目级配置**：按项目需求调整规则

### 插件使用建议

1. **从简单开始**：先使用声明式插件
2. **测试充分**：在测试环境验证插件
3. **性能考虑**：WASM 插件性能更好

## 故障排除

### 规则不触发

1. 检查规则是否启用
2. 确认流量类型匹配
3. 查看日志中的扫描信息

### 插件加载失败

1. 检查 manifest.toml 格式
2. 确认 WASM 文件完整
3. 点击"重载插件"刷新

### 性能问题

1. 减少同时启用的规则数量
2. 使用 WASM 插件替代复杂规则
3. 调整扫描触发时机
