# 报告导出

FlowMind 支持将测试结果导出为结构化报告，便于记录、分享和归档。

## 报告格式

| 格式 | 说明 | 适用场景 |
|------|------|----------|
| JSON | 结构化数据格式 | 归档、二次整理（在应用内导出） |
| PDF | 可打印文档格式 | 正式报告、归档 |

## 导出报告

### 导出步骤

1. 确保当前项目有测试数据
2. 进入 **报告** 页面
3. 选择导出格式
4. 配置报告选项
5. 点击 **导出**

### 报告选项

| 选项 | 说明 |
|------|------|
| 包含 Flow | 是否包含流量数据 |
| 包含 Finding | 是否包含安全发现 |
| 包含素材 | 是否包含素材剪藏 |
| 时间范围 | 导出指定时间段的数据 |

## JSON 报告

导出文件包含项目元信息、统计摘要，以及可选的 Flow、Finding、素材等章节。具体字段以当前版本导出结果为准，**完整 JSON Schema 不向公开文档提供**；如需企业对接请联系技术支持。

### 使用场景

- 团队内归档与备份
- 配合内部流程做人工复核

## PDF 报告

### 报告内容

PDF 报告包含以下章节：

1. **封面**：项目名称、日期、版本
2. **摘要**：测试概览、统计数据
3. **安全发现**：按严重程度排序的 Finding 列表
4. **流量统计**：请求方法、状态码分布
5. **附录**：详细数据表格

### 报告样式

- 专业排版
- 表格和图表
- 页眉页脚
- 目录导航

## 素材剪藏

### 什么是素材剪藏

素材剪藏用于收集和整理测试过程中的重要信息：

- 有趣的请求/响应
- 关键技术细节
- 测试笔记
- 截图和注释

### 添加素材

1. 在转发器选择请求
2. 右键 → **添加到素材**
3. 添加备注说明

### 管理素材

进入 **报告** → **素材管理**：

- 查看所有素材
- 编辑备注
- 删除素材
- 导出素材

## 备忘录

### 功能说明

备忘录用于记录测试过程中的想法、发现和计划：

- 测试思路
- 待验证的猜想
- 重要发现
- 后续计划

### 使用备忘录

1. 进入 **报告** → **备忘录**
2. 点击 **新建备忘录**
3. 输入标题和内容
4. 保存

## 报告模板

### 自定义模板

支持自定义报告模板：

1. 创建模板文件（JSON 格式）
2. 配置模板结构
3. 在导出时选择模板

### 内置模板

| 模板 | 说明 |
|------|------|
| 标准报告 | 完整测试报告 |
| 摘要报告 | 简化版报告 |
| 技术报告 | 详细技术数据 |

## 已导出报告

### 查看历史

进入 **报告** → **已导出**：

- 查看导出历史
- 重新下载报告
- 删除旧报告

### 报告管理

| 操作 | 说明 |
|------|------|
| 下载 | 重新下载报告文件 |
| 删除 | 删除报告记录 |
| 查看详情 | 查看导出配置 |

## 使用建议

### 报告内容

1. **包含关键发现**：确保重要 Finding 在报告中
2. **添加上下文**：使用素材和备注补充说明
3. **定期导出**：避免数据丢失

### 报告格式

1. **正式场合**：使用 PDF 格式
2. **技术交流**：使用 JSON 格式
3. **内部记录**：两种格式都导出

### 报告质量

1. **检查数据完整性**：确认所有数据都包含
2. **验证格式正确**：预览报告内容
3. **保护敏感信息**：注意报告中的敏感数据

## 故障排除

### 导出失败

1. 检查磁盘空间
2. 确认项目有数据
3. 查看错误日志

### 报告内容缺失

1. 检查导出选项配置
2. 确认时间范围设置
3. 验证数据是否存在

### 格式问题

1. PDF 样式异常：检查字体和样式配置
2. JSON 格式错误：使用 JSON 验证工具检查
