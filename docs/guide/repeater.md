# 重发器

重发器用于对历史请求进行重放或手动修改后重新发送，是安全测试和 API 调试的重要工具。

![重发器](/screenshots/重发器.png)

## 功能概述

- **请求重放**：基于已有请求快速重放
- **原始报文编辑**：直接编辑 HTTP 原始报文
- **结构化编辑**：可视化编辑请求各部分
- **重发历史**：保存请求标签，方便反复测试

## 发送请求到重发器

### 从转发器发送

1. 在转发器列表中选择请求
2. 右键 → **发送到重发器**
3. 请求自动加载到新标签页

### 从模糊器发送

1. 在模糊器结果中选择请求
2. 右键 → **发送到重发器**

### 手动创建

1. 点击标签栏的 **+** 按钮
2. 手动输入请求内容

## 编辑请求

### 原始模式

直接编辑 HTTP 原始报文：

```http
GET /api/users HTTP/1.1
Host: example.com
Authorization: Bearer token123
Accept: application/json

```

### 结构化模式

可视化编辑各个部分：

| 部分 | 可编辑内容 |
|------|-----------|
| 请求行 | 方法、路径、协议版本 |
| 请求头 | 添加、修改、删除请求头 |
| 请求体 | JSON、表单、原始内容 |
| 连接信息 | Host、Port、TLS 开关 |

## 发送请求

### 基本发送

1. 编辑完成请求内容
2. 点击 **发送** 按钮
3. 响应显示在右侧面板

### 连接配置

| 配置项 | 说明 |
|--------|------|
| Host | 目标服务器地址 |
| Port | 目标服务器端口 |
| TLS | 是否使用 HTTPS |

::: tip 提示
连接配置会覆盖请求头中的 Host 值，用于测试不同环境。
:::

## 响应查看

### 响应概览

- 状态码
- 响应大小
- 响应时间

### 响应头

完整的响应头列表

### 响应体

支持多种格式查看：

- **Pretty**：格式化显示（JSON、XML 等）
- **Raw**：原始内容
- **Hex**：十六进制视图

## 标签管理

### 保存标签

1. 编辑好请求后
2. 点击 **保存** 按钮
3. 输入标签名称

### 管理标签

| 操作 | 说明 |
|------|------|
| 重命名 | 右键标签 → 重命名 |
| 删除 | 右键标签 → 删除 |
| 复制 | 右键标签 → 复制 |

## 重发历史

每次发送请求都会记录：

- 发送时间
- 请求内容快照
- 响应结果

可以在历史记录中查看和重用之前的请求。

## 使用场景

### API 调试

```
1. 从转发器捕获 API 请求
2. 发送到重发器
3. 修改请求参数
4. 重新发送，观察响应
5. 反复调试直到正确
```

### 认证测试

```
1. 捕获需要认证的请求
2. 修改 Authorization 头
3. 测试不同 Token 的权限
4. 验证认证机制
```

### 参数篡改

```
1. 捕获包含业务参数的请求
2. 修改参数值
3. 观察响应变化
4. 识别潜在漏洞
```

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl/⌘ + Enter` | 发送请求 |
| `Ctrl/⌘ + S` | 保存标签 |
| `Ctrl/⌘ + N` | 新建标签 |
| `Ctrl/⌘ + W` | 关闭当前标签 |
