# AI 功能

FlowMind 是一个 AI-Native 安全平台，集成了多种 AI 能力帮助安全测试人员更高效地发现和验证漏洞。

![AI-会话](/screenshots/AI-会话.png)

## 功能概览

| 功能 | 说明 |
|------|------|
| AI 聊天 | 多 Provider 对话，支持流式输出 |
| 工具调用 | AI 可调用内置工具查询 Flow、Finding 等 |
| 知识库 | RAG 检索，导入文档建立知识库 |
| 安全记忆 | 自动记录目标特征、漏洞、攻击模式 |
| Agent 分析 | 多 Agent 协作进行安全分析 |
| 攻击图谱 | 可视化攻击路径和风险传播 |

## AI 页面

进入 **AI** 页面，包含以下标签：

| 标签 | 功能 |
|------|------|
| 上下文 | AI 对话上下文配置 |
| 记忆 | 安全记忆系统 |
| 知识库 | 知识库管理 |
| 会话 | AI 对话会话 |
| Agents | 多 Agent 安全分析 |
| Attack Map | 攻击图谱可视化 |

## AI 聊天

### 配置 Provider

进入 **设置 → AI**：

1. 在 **对话模型** 区域选择活跃 Provider 与模型
2. 点击 **Provider** 卡片，在弹窗中填写 API Key、Base URL 与可用模型列表
3. 可选：配置 **Embedding**、**MCP Servers**、**AI 角色** 等扩展项
4. 点击页面右上角 **保存设置**

详细说明见 [设置](./settings.md#ai-配置)。

### 发送消息

1. 在聊天框输入问题
2. 点击发送或按 Enter
3. AI 流式返回响应

### 发送 Flow 到 AI

1. 在转发器选择请求
2. 右键 → **发送到 AI**
3. AI 自动分析该请求

## 工具调用

AI 可在对话中**自动调用应用内置能力**（如查询流量、查看发现、解码令牌、触发扫描等），无需您手动复制数据。具体可用能力取决于当前 Skill、角色与许可范围，以界面展示为准。

### 示例用法（自然语言）

在会话中直接描述需求即可，例如：

- 「分析当前项目里有哪些 API 端点」
- 「总结最近的高危发现」
- 「解读这条请求里的 JWT 载荷」

AI 会在需要时自行调用相应能力并汇总结果。

## 安全记忆

应用会按**项目**记录测试过程中有价值的信息（目标特征、资产、身份、发现与攻击路径等），供后续对话检索。可在 **AI → 记忆** 中浏览与管理。

启用 Embedding 后，记忆与知识库检索效果更好（见 [设置](./settings.md)）。

## Agent 分析

在 **AI → Agents**（或相关入口）可发起多步骤安全分析任务，例如流量模式梳理、认证机制审查、漏洞线索汇总、攻击路径推演等。在聊天中用自然语言说明分析目标即可。

示例：

- 「请对当前项目做一次综合安全分析」
- 「重点看看认证与权限相关的问题」

## 攻击图谱

### 节点类型

| 节点类型 | 颜色 | 说明 |
|----------|------|------|
| Endpoint | 蓝色 | API 端点 |
| Identity | 紫色 | 用户/身份 |
| Credential | 黄色 | Token/凭证 |
| Vulnerability | 红色 | 漏洞 |
| Data | 绿色 | 数据/资源 |

### 边类型

| 边类型 | 说明 |
|--------|------|
| CanAccess | 可以访问 |
| Exploits | 利用关系 |
| LeaksTo | 泄露到 |
| EscalatesTo | 提权到 |

### 使用方式

1. 切换到 **Attack Map** 标签
2. 查看节点和边的可视化
3. 点击节点查看详细信息
4. 查看攻击路径建议

## 智能重放与 Fuzz 建议

AI 可结合当前 Flow 给出**重放建议**或**模糊测试思路**（如参数变异、权限边界探测）。在 Repeater / Fuzzer 模块中执行具体操作；向 AI 描述端点与测试目标即可获取策略建议。

## 最佳实践

1. **先让 AI 理解目标**：例如「分析当前流量，了解架构和认证方式」
2. **结合记忆与知识库**：导入测试文档或规范到知识库，提问时引用项目上下文
3. **分析 → 图谱 → 验证**：先用 Agent/对话形成假设，用 Attack Map 对照路径，再在 Repeater/Fuzzer 中验证
4. **记录结论**：将重要发现通过报告或项目内素材归档

## 配置选项

### Provider 配置

| 选项 | 说明 |
|------|------|
| Provider | AI 服务提供商 |
| API Key | 认证密钥 |
| Model | 使用的模型 |
| Temperature | 创造性参数 |

### 上下文配置

| 选项 | 说明 |
|------|------|
| Max Tokens | 最大 Token 数 |
| Context Window | 上下文窗口大小 |
| Include Flows | 是否包含 Flow 数据 |
| Include Findings | 是否包含 Finding 数据 |
