# 项目管理

项目管理功能用于隔离不同目标的测试数据，便于组织和管理安全测试工作。

## 功能概述

- **项目隔离**：每个项目的 Flow、Finding、设置独立
- **快速切换**：在不同项目间快速切换上下文
- **项目设置**：按项目配置扫描规则、AI 参数等

## 创建项目

### 新建项目

1. 点击标题栏的项目下拉菜单
2. 选择 **新建项目**
3. 输入项目名称和描述
4. 点击 **创建**

### 项目信息

| 字段 | 说明 |
|------|------|
| 名称 | 项目显示名称 |
| 描述 | 项目描述（可选） |
| 创建时间 | 自动记录 |
| 最后打开 | 最后一次打开的时间 |

## 项目切换

### 切换项目

1. 点击标题栏的项目下拉菜单
2. 从列表中选择项目
3. 项目上下文自动切换

### 切换影响

切换项目时，以下内容会随之切换：

- 转发器 Flow 列表
- 扫描器 Finding 列表
- 项目配置（扫描规则、AI 设置）
- 重发器标签页（可选）

## 项目设置

进入 **设置** → **项目设置**：

### 扫描规则配置

选择当前项目启用的扫描规则：

- [x] 敏感信息泄露
- [x] Cookie 安全
- [ ] 传输安全
- [x] CORS 配置
- ...

### AI 配置

按项目配置 AI 参数：

| 配置项 | 说明 |
|--------|------|
| 默认 Provider | 项目使用的 AI Provider |
| 上下文预算 | AI 分析的上下文大小 |
| 自动记忆 | 是否自动记录到记忆系统 |

## 数据隔离

### Flow 隔离

- 新捕获的 Flow 自动绑定当前项目
- 转发器列表只显示当前项目的 Flow
- 切换项目后 Flow 列表自动刷新

### Finding 隔离

- 扫描发现的 Finding 绑定当前项目
- Finding 列表按项目过滤
- 项目删除时级联删除关联 Finding

### 设置隔离

- 扫描规则配置按项目保存
- AI 配置按项目保存
- 其他设置为全局共享

## 项目操作

### 编辑项目

1. 进入 **设置** → **项目设置**
2. 修改项目名称或描述
3. 点击 **保存**

### 删除项目

1. 进入 **设置** → **项目设置**
2. 点击 **删除项目**
3. 确认删除

::: warning 危险操作
删除项目会级联删除：
- 所有关联的 Flow 数据
- 所有关联的 Finding 数据
- 项目配置

此操作不可撤销！
:::

### 导出项目

支持导出项目数据：

- Flow 数据（JSON 格式）
- Finding 数据（JSON 格式）
- 项目配置

## 使用场景

### 多目标测试

```
场景：同时测试多个目标

1. 为目标 A 创建项目 A
2. 为目标 B 创建项目 B
3. 切换到项目 A，配置代理，开始测试
4. 切换到项目 B，继续测试
5. 数据完全隔离，互不干扰
```

### 团队协作

```
场景：多人测试同一目标

1. 创建共享项目
2. 各自导出测试结果
3. 合并结果到统一报告
```

### 版本管理

```
场景：跟踪测试进度

1. 为每个测试阶段创建项目
2. 记录每个阶段的发现
3. 对比不同阶段的进展
```

## 最佳实践

1. **按目标创建项目**
   - 每个独立目标一个项目
   - 避免数据混乱

2. **命名规范**
   - 使用清晰的项目名称
   - 包含目标信息或日期

3. **定期清理**
   - 删除不再需要的项目
   - 释放存储空间

4. **备份重要数据**
   - 导出重要项目的报告
   - 保留测试证据
