# 模糊器

模糊器用于对 HTTP 请求进行受控变异测试，自动发现潜在的安全漏洞。

## 功能概述

- **多策略变异**：支持 IDOR、Auth Strip、Header 词表等策略
- **并发控制**：可配置并发数和请求速率
- **结果筛选**：按状态码、长度、关键词筛选结果
- **基线对比**：自动与原始请求对比差异

## 发送请求到模糊器

### 从转发器发送

1. 在转发器列表中选择请求
2. 右键 → **发送到模糊器**

### 从重发器发送

1. 在重发器中编辑好请求
2. 右键 → **发送到模糊器**

### 手动粘贴

1. 进入模糊器页面
2. 粘贴原始 HTTP 请求
3. 填写连接信息（Host、Port、TLS）

## 变异策略

### IDOR 数字变异

识别 URL 中的数字参数，生成相邻值变体：

```
原始: /api/users/123
变异: /api/users/122, /api/users/124, /api/users/100, ...
```

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 变异范围 | ±N 的范围 | 10 |
| 最大 Case 数 | 最大变异数量 | 100 |

### Auth Strip 认证剥离

移除或清空认证相关的请求头：

```
移除: Authorization
移除: Cookie
清空: X-API-Key
```

### Header 词表注入

对指定请求头注入预定义词表：

```
词表: admin, test, debug, backup, ...
目标: X-Forwarded-For, X-Real-IP
```

### 组合策略

支持多个策略组合使用：

```
策略 1: IDOR 数字变异（路径参数）
策略 2: Auth Strip（移除 Authorization）
```

## 任务配置

### 基本配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 并发数 | 同时发送的请求数 | 3 |
| 请求间隔 | 两次请求之间的间隔 | 100ms |
| 单请求超时 | 单个请求的超时时间 | 10s |
| 失败重试 | 网络失败时的重试次数 | 0 |

### 速率限制

| 模式 | 说明 |
|--------|------|
| 令牌桶 | 平滑控制请求速率 |
| 固定间隔 | 固定时间间隔发送 |

### 基线测试

- [x] 先发送原始请求作为基线
- [x] 结果自动与基线对比

## 执行任务

### 启动任务

1. 配置好策略和参数
2. 点击 **开始** 按钮
3. 任务开始执行

### 任务状态

| 状态 | 说明 |
|------|------|
| 待运行 | 任务已创建，等待执行 |
| 运行中 | 正在发送请求 |
| 已完成 | 所有请求已发送完毕 |
| 已取消 | 用户手动取消 |
| 失败 | 执行出错 |

### 取消任务

点击 **取消** 按钮：

- 已排队的请求不再发送
- 正在发送的请求等待完成
- 结果标记为"已取消"

## 结果分析

### 结果列表

| 字段 | 说明 |
|------|------|
| Case ID | 变异用例编号 |
| 状态码 | HTTP 响应状态码 |
| 响应长度 | 响应体大小 |
| 耗时 | 请求耗时 |
| 差异 | 与基线的差异标记 |

### 筛选功能

| 筛选维度 | 说明 |
|----------|------|
| 状态码 | 按状态码筛选（如 401、403、500） |
| 响应长度 | 按长度范围筛选 |
| 关键词 | 响应体包含特定关键词 |
| 耗时 | 按耗时范围筛选 |

### 差异高亮

自动高亮以下差异：

- 状态码变化
- 响应长度变化超过 10%
- 响应时间异常

## 结果处理

### 查看详情

点击结果行查看完整请求/响应

### 发送到重发器

将可疑请求发送到重发器进一步验证

### 创建 Finding

将发现的漏洞创建为 Finding 记录

### 导出结果

支持导出为 JSON/CSV 格式

## 使用场景

### IDOR 测试

```
1. 捕获 /api/users/{id} 请求
2. 选择 IDOR 数字变异策略
3. 配置变异范围
4. 执行任务
5. 筛选 200 状态码的结果
6. 验证是否能访问其他用户数据
```

### 认证绕过测试

```
1. 捕获需要认证的请求
2. 选择 Auth Strip 策略
3. 执行任务
4. 观察移除认证后的响应
5. 筛选非 401/403 的结果
```

### 参数注入测试

```
1. 捕获包含自定义头的请求
2. 选择 Header 词表策略
3. 配置目标头和词表
4. 执行任务
5. 分析响应差异
```

## 注意事项

::: warning 负责任使用
- 仅对授权的目标进行测试
- 控制请求速率，避免对目标造成负担
- 遵守相关法律法规
:::

::: tip 性能建议
- 并发数建议从 3 开始
- 设置合理的请求间隔
- 大量结果时使用筛选功能
:::
