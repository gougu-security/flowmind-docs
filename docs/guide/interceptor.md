# 拦截器

拦截器允许您实时拦截、修改和控制在途的 HTTP 请求。

## 功能概述

拦截器提供以下能力：

- **Hold**：暂停请求，等待手动放行
- **Modify**：在放行前修改请求内容
- **Drop**：丢弃请求，不发送到目标服务器

## 启用拦截器

1. 进入 **拦截器** 页面
2. 点击 **启用拦截** 开关
3. 配置拦截规则（可选）

## 拦截规则

### 按方法拦截

选择要拦截的 HTTP 方法：

- [x] GET
- [x] POST
- [ ] PUT
- [ ] DELETE
- [ ] 其他

### 按域名拦截

配置要拦截的域名列表：

```
api.example.com
*.internal.company.com
```

### 按路径拦截

使用正则表达式匹配路径：

```
^/api/v1/users
^/admin/
```

## 拦截队列

当请求被拦截时，会进入拦截队列：

| 字段 | 说明 |
|------|------|
| 方法 | HTTP 方法 |
| URL | 请求 URL |
| 状态 | 等待中 / 已修改 / 已放行 / 已丢弃 |
| 拦截时间 | 请求被拦截的时间 |

## 操作请求

### 查看详情

点击队列中的请求，查看完整详情：

- 请求头
- 请求体
- 原始报文

### 修改请求

1. 选中要修改的请求
2. 点击 **编辑** 按钮
3. 修改请求内容：
   - 修改 URL
   - 修改请求头
   - 修改请求体
4. 点击 **保存修改**

### 放行请求

- **放行**：将请求（可能已修改）发送到目标服务器
- **批量放行**：选中多个请求，一次性放行

### 丢弃请求

- **丢弃**：丢弃请求，客户端将收到连接超时或错误
- **批量丢弃**：选中多个请求，一次性丢弃

## 使用场景

### 安全测试

```
场景：测试 API 参数篡改

1. 启用拦截器
2. 触发目标请求
3. 在拦截队列中修改请求参数
4. 放行修改后的请求
5. 观察响应结果
```

### 调试开发

```
场景：调试前端请求

1. 按域名拦截目标 API
2. 查看请求详情
3. 修改请求头添加调试信息
4. 放行请求
```

### 流量控制

```
场景：阻止特定请求

1. 配置路径拦截规则
2. 匹配的请求自动进入队列
3. 丢弃不需要的请求
```

## 注意事项

::: warning 性能影响
启用拦截器会显著降低网络性能，仅在需要时启用。
:::

::: tip 自动超时
被拦截的请求会在 60 秒后自动超时，请及时处理队列中的请求。
:::
